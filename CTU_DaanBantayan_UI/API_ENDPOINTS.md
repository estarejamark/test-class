-## Authentication
-
-### Login
-
-- **Method:** POST
-- **Path:** `/api/auth/session`
-- **Description:** Authenticate user and receive JWT cookie.
-- **Request Payload:**
-
-```json
-{
-  "email": "user@example.com",
-  "password": "secure123"
-}
-```
-
-- **Response:**
-  - **200 OK**
-  - Sets `jwt` cookie
-  - Body:
-
-```json
-{
-  "userResponse": {
-    /* user info */
-  },
-  "authorization": {
-    /* token info */
-  }
-}
-```
-
-- **Edge Cases:**
-  - Returns 401 for invalid credentials
-  - Rate limiting may apply
-
-### Refresh Token
-
-- **Method:** POST
-- **Path:** `/api/auth/refresh?userId=UUID`
-- **Description:** Refresh JWT token for a user.
-- **Request Payload:** None (userId as query param)
-- **Response:**
-  - **200 OK**
-  - Sets new `jwt` cookie
-  - Body: new JWT string
-- **Edge Cases:**
-  - 400 if userId missing/invalid
-  - 401 if refresh fails
-
-### Logout
-
-- **Method:** POST
-- **Path:** `/api/auth/logout`
-- **Description:** Invalidate JWT and clear cookie.
-- **Request Payload:** None
-- **Response:**
-  - **200 OK**
-  - Body: `"Logged out successfully"`
-- **Edge Cases:**
-  - Requires valid `jwt` cookie
-

-## Users
-
-### Register User
-
-- **Method:** POST
-- **Path:** `/api/users`
-- **Description:** Register a new user (admin only).
-- **Request Payload:**
-
-```json
-{
-  "email": "user@example.com",
-  "password": "secure123",
-  "role": "STUDENT" // or "TEACHER", "ADMIN"
-}
-```
-
-- **Response:**
-  - **201 Created**
-  - Body: user ID or success message
-- **Edge Cases:**
-  - Requires valid admin `jwt` cookie
-  - 400 if email already exists
-
-### Get All Teachers
-
-- **Method:** GET
-- **Path:** `/api/users/teachers`
-- **Description:** List all teacher profiles.
-- **Response:**
-  - **200 OK**
-  - Body: array of teacher profiles
-- **Edge Cases:**
-  - Requires valid `jwt` cookie
-
-### Update User Status
-
-- **Method:** PUT
-- **Path:** `/api/users?id=UUID`
-- **Description:** Update user status (e.g., activate/deactivate).
-- **Request Payload:** None (id as query param)
-- **Response:**
-  - **200 OK**
-  - Body: success message
-- **Edge Cases:**
-  - Requires valid `jwt` cookie
-  - 400 if id missing/invalid
-

-## Profiles
-
-### Get My Profile
-
-- **Method:** GET
-- **Path:** `/api/profiles/me`
-- **Description:** Get the profile of the authenticated user.
-- **Response:**
-  - **200 OK**
-  - Body: profile object
-- **Edge Cases:**
-  - Requires valid `jwt` cookie
-
-### Create Profile
-
-- **Method:** POST
-- **Path:** `/api/profiles`
-- **Description:** Create a profile for the authenticated user.
-- **Request Payload:**
-
-```json
-{
-  "firstName": "John",
-  "middleName": "A.",
-  "lastName": "Doe",
-  "gender": "MALE", // or "FEMALE", "OTHER"
-  "birthDate": "1990-01-01",
-  "contactNumber": "+1234567890",
-  "address": "123 Main St"
-}
-```
-
-- **Response:**
-  - **201 Created**
-  - Body: profile ID or success message
-- **Edge Cases:**
-  - Requires valid `jwt` cookie
-  - 400 for missing required fields
-
-### Update Profile
-
-- **Method:** PUT
-- **Path:** `/api/profiles/{id}`
-- **Description:** Update the profile of the authenticated user.
-- **Request Payload:** Same as Create Profile
-- **Response:**
-  - **201 Created**
-  - Body: profile ID or success message
-- **Edge Cases:**
-  - Requires valid `jwt` cookie
-  - 400 for missing/invalid fields
-
-### Search Profiles
-
-- **Method:** GET
-- **Path:** `/api/profiles`
-- **Description:** Search/filter profiles with pagination.
-- **Query Params:**
-  - `role` (optional): `ADMIN`, `TEACHER`, `STUDENT`
-  - `name` (optional): string
-  - `page` (default 0): int
-  - `size` (default 10): int
-- **Response:**
-  - **200 OK**
-  - Body: paginated list of profiles
-- **Edge Cases:**
-  - Requires valid `jwt` cookie
-

-## Records
-
-### Get Pending Packages
-
-- **Method:** GET
-- **Path:** `/api/records/pending`
-- **Description:** List all pending quarter packages for review.
-- **Response:**
-  - **200 OK**
-  - Body: array of quarter package objects
-
-### Get Published Packages
-
-- **Method:** GET
-- **Path:** `/api/records/published`
-- **Description:** List all published quarter packages.
-- **Response:**
-  - **200 OK**
-  - Body: array of quarter package objects
-
-### Get Package Details
-
-- **Method:** GET
-- **Path:** `/api/records/{sectionId}/{quarter}`
-- **Description:** Get details of a specific quarter package.
-- **Path Params:**
-  - `sectionId`: UUID of the section
-  - `quarter`: Quarter enum (Q1, Q2, Q3, Q4)
-- **Response:**
-  - **200 OK**
-  - Body: quarter package object
-- **Edge Cases:**
-  - 404 if package not found
-
-### Approve Package
-
-- **Method:** PATCH
-- **Path:** `/api/records/{packageId}/approve`
-- **Description:** Approve a pending quarter package.
-- **Path Params:**
-  - `packageId`: UUID of the package
-- **Request Headers:**
-  - `Authorization`: Bearer token
-- **Response:**
-  - **200 OK**
-  - Body: updated quarter package object
-- **Edge Cases:**
-  - 400 if package not in approvable state
-
-### Return Package
-
-- **Method:** PATCH
-- **Path:** `/api/records/{packageId}/return`
-- **Description:** Return a package with remarks for revision.
-- **Path Params:**
-  - `packageId`: UUID of the package
-- **Query Params:**
-  - `remarks`: String with return remarks
-- **Request Headers:**
-  - `Authorization`: Bearer token
-- **Response:**
-  - **200 OK**
-  - Body: updated quarter package object
-
-### Publish Package
-
-- **Method:** PATCH
-- **Path:** `/api/records/{packageId}/publish`
-- **Description:** Publish an approved package (admin only).
-- **Path Params:**
-  - `packageId`: UUID of the package
-- **Request Headers:**
-  - `Authorization`: Bearer token
-- **Response:**
-  - **200 OK**
-  - Body: updated quarter package object
-- **Edge Cases:**
-  - 403 if not admin
-  - 400 if package not approved
-
-### Get Package Approvals
-
-- **Method:** GET
-- **Path:** `/api/records/{packageId}/approvals`
-- **Description:** Get approval history for a package.
-- **Path Params:**
-  - `packageId`: UUID of the package
-- **Response:**
-  - **200 OK**
-  - Body: array of approval records
-

-## General Notes
-
-- All authenticated endpoints require a valid `jwt` cookie.
-- All dates/times use ISO 8601 format.
-- All UUIDs must be valid.
-- Validation errors return 400 with error details.
-- Unauthorized access returns 401.
-- Not found returns 404.
-- Server errors return 500.
